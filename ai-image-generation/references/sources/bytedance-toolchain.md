# 方向1：豆包在短剧创作中的角色

## 核心定位

豆包在短剧/漫剧工作流中**不承担最终「生图渲染」的主力角色**，而是作为**文本中枢**与**角色/分镜资产孵化器**：

1. **写剧本**：生成带强冲突、快节奏、1–3 分钟的短剧故事。  
2. **拆分镜**：把剧本拆解为「镜头序号 + 景别 + 画面描述 + 对白 + 时长」的表格。  
3. **定角色**：输出详细到可供 AI 生图使用的角色设定（发型、服饰、标志性特征）。  
4. **批量生成角色定妆照/分镜图**：在豆包 AI 生图界面中，以角色设定为 Prompt，生成「角色种子图」，再供后续即梦/豆包自身生图时做参考图。

> 来源 [知乎全流程指南](https://zhuanlan.zhihu.com/p/2008560818791407761) 将流程总结为：豆包（故事+分镜）→ 即梦 Seedance 2.0 Fast（视频渲染）→ 剪映（后期优化）。  
> 来源 [今日头条教程](https://www.toutiao.com/article/7631771187063882240/) 将流程总结为：DeepSeek 写剧本 + 豆包定角色 + 即梦生分镜 + 剪映剪辑。

## 豆包是否支持生图？是否支持分镜图？

| 能力 | 是否支持 | 说明 |
|------|----------|------|
| 文生图 | ✅ 支持 | 豆包 AI 生图界面可生成角色定妆照、分镜候选图。 |
| 图生图/参考图 | ✅ 支持 | 可上传角色定妆照作为参考图，批量生成同角色不同场景分镜。 |
| 分镜图自动排版 | ⚠️ 半支持 | 豆包本身不输出「分镜格」，需把它的分镜描述复制到生图工具。 |
| 角色一致性锁定 | ✅ 支持 | 通过参考图 + 提示词首句重申「保持长相完全一致」实现。 |

> 来源 [今日头条教程](https://www.toutiao.com/article/7631771187063882240/) 强调：豆包 Seedance 2.0 版本通过「特征锁定算法」使角色服装、发型在不同镜头保持稳定。

## 豆包 + 即梦 + 剪映协同工作流（典型链路）

```
剧本/故事  →  分镜脚本  →  角色定妆照  →  批量分镜图  →  图生视频  →  剪辑合成
(豆包文本)    (豆包文本)    (豆包生图)     (豆包/即梦)     (即梦)      (剪映)
```

1. **豆包写剧本**：Prompt 结构建议为「题材 + 时长 + 人物设定 + 核心冲突 + 转折要求」。  
2. **豆包拆分为分镜脚本**：Prompt 要求每 2 秒一个镜头，标注镜头序号、时长、景别、画面内容、人物动作、台词、音效。  
3. **豆包生成角色定妆照**：基于 DeepSeek/豆包输出的角色设定，生成每个角色的正面高清图。  
4. **豆包批量生成分镜图**：上传角色定妆照作为参考图，结合分镜画面描述逐一生成。  
5. **即梦图生视频**：把分镜图作为首帧，生成 3–6 秒动态镜头。  
6. **剪映合成**：剪辑、AI 配音、字幕、音效、转场、调色、导出。

> 来源 [CSDN 漫剧教程](https://blog.csdn.net/seoread/article/details/159682536) 将漫剧流程概括为：豆包孵化剧本 → 豆包拆解分镜 → 生成角色三视图 → 绘制镜头 → 即梦 Seedance 2.0 唤醒动态 → 剪辑包装。

## 最佳实践

- **角色设定必须具体到「可生图」**：包含发型、服饰、标志性道具、年龄、气质。例如「干练短发、白色衬衫+高腰西裤、戴银色细框眼镜」。  
- **分镜描述要包含构图、动作、光影、氛围**：避免写「两人对话」，要写「林悦抬头望向张组长，眼神从紧张转为坚定，张组长站在会议桌对面双手抱胸冷笑」。  
- **同场景镜头统一关键词**：如所有办公室场景都使用「现代化办公室，落地窗，灰色办公桌椅」。  
- **固定光线描述**：每个同场景镜头使用相同光线词，如「柔和自然光」或「暖色顶光」，避免光影跳跃。  
- **保存命名规范**：按「集数_镜头序号」命名，如 `E01_S01.jpg`，方便后续剪辑定位。  

## 避坑

- **未锁定参考图导致「换镜头换脸」**：每个分镜生成前必须上传角色定妆照，并在提示词开头重申「保持长相完全一致」。  
- **多人镜头角色错乱**：分别上传多人参考图，并在提示词中明确标注位置关系，如「画面左侧为林悦，右侧为张组长」。  
- **场景描述差异导致风格不统一**：在剧本阶段就固定场景模板，不要任由 AI 自由发挥。  
- **手指等细节问题**：AI 画手是普遍难点。提示词中写「手部自然放在键盘上」或「双手持文件遮挡」，避免「手指张开」等复杂描述。

# 方向2：即梦（Dreamina / Jimeng）生图实战

## 核心定位

即梦是字节跳动旗下的一站式 AI 创作平台，在短剧/漫剧工作流中承担**主力生图 + 图生视频**角色：

- **文生图**：根据分镜描述生成静态分镜图。  
- **图生图/参考图**：通过「角色特征」「参考图」保持人物一致性。  
- **智能画布**：多模态编辑、局部重绘、换装、换背景。  
- **图生视频**：Seedance 2.0 / 2.0 Fast 把静态分镜转为动态镜头。  
- **多图参考/角色 ID**：支持 2–6 张参考图融合，或训练角色 ID 保持跨镜头一致性。

> 来源 [百度百科](https://baike.baidu.com/item/%E5%8D%B3%E6%A2%A6AI/64609531) 说明：即梦 AI 整合文生图、图生图、文生视频及图生视频等核心功能，依托自研 Seedream 和 Seedance 模型架构。  
> 来源 [CSDN 对比](https://blog.csdn.net/u014177256/article/details/157644721) 指出：即梦 AI 在「角色 ID 机制、首帧+尾帧+中间补帧、三维锚定」上保持跨镜头一致性表现突出。

## 角色一致性 / 智能参考功能

| 功能 | 操作路径 | 关键参数 | 适用场景 |
|------|----------|----------|----------|
| 角色特征 | 生图页 → 上传参考图 → 勾选「角色特征」 | 参考强度 70%（推荐）、脸部参考/主体参考 | 同角色不同场景、不同表情 |
| 人脸写真 | 参考图 → 选择「人脸写真」 | 选择 3.0/4.0 模型 | 保持面部一致性 |
| 4.0 局部锁定 | 上传参考图 → 手动涂抹面部/区域 → 生成 | 4.0 模型 + 笔刷/框选 | 高精准面部一致性 |
| 多图参考融合 | 上传 2–6 张参考图 → 输入动作描述 | 角色动作跟随提示词 | 多人同框、互动动作 |
| 角色 ID / 三维锚定 | 生成满意图后保存为角色 ID | 跨镜头复用 | 连续剧、漫剧系列 |

> 来源 [知乎教程](https://zhuanlan.zhihu.com/p/1890369960041228029) 建议：参考强度 70% 既能保持五官一致，又不至于让动作僵硬；高于 80% 易姿势呆板，低于 50% 易五官变形。  
> 来源 [知乎 4.0 教程](https://zhuanlan.zhihu.com/p/1951239394070605930) 说明：即梦 4.0 支持一次生成 14 张系列图、2–6 张参考图融合、一图多视角、一句话换背景/换衣服/局部重绘。

## 中文 Prompt 写法建议

1. **避免抽象形容词**：少用「可爱」「漂亮」，多用肉眼可见的细节，如「圆脸杏眼、腮红明显、双丸子头、黄色汉服」。  
2. **结构公式**：
   ```
   [风格] + [主体] + [服饰/发型/道具] + [动作/表情] + [场景] + [光影] + [镜头/构图] + [质量词]
   ```
3. **示例**：
   - 角色定妆：`3D皮克斯风格，古代小女孩，黄色汉服，双丸子头，圆脸杏眼，腮红明显，皮肤通透有光泽，浅景深虚化背景`
   - 分镜图：`电影级构图，极具张力的双人侧景对峙镜头，侧逆光打在两人脸上，深夜暗殿，清冷月光，暖色烛光，背景微微虚化`
4. **动态提示词公式（用于图生视频）**：
   ```
   [主体动作] + [环境变化] + [镜头运动] + [风格限定] + [时长] + [负面提示词]
   ```

## 分镜图、连续画面、同场景多角度

| 需求 | 实现方式 | 关键技巧 |
|------|----------|----------|
| 分镜图 | 文生图/图生图，按镜头描述逐一生成 | 上传角色参考图，保持人物一致 |
| 连续画面 | 使用 4.0「一次生成 14 张系列图」 | 提示词中描述序列动作或场景变化 |
| 同场景多角度 | 上传一张参考图，输入「生成不同视角的人物场景」 | 即梦 4.0 可输出正面、侧面、45°角，且服饰/表情一致 |
| 同场景多人对话 | 分别上传多人参考图，明确位置关系 | 提示词中标注「画面左侧为 A，右侧为 B」 |

> 来源 [知乎 4.0 教程](https://zhuanlan.zhihu.com/p/1951239394070605930)：上传一张模特街头图，输入「生成不同视角的人物场景」，AI 可出正面、侧面、45°角，耳环细节保持一致。

## 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 脸崩/五官变形 | 参考强度过低、未上传参考图、模型版本过旧 | 使用 4.0 模型；参考强度 70%–80%；上传高清面部参考图；用局部锁定涂抹面部 |
| 手崩/手指畸形 | AI 画手是通用难点；提示词手部描述复杂 | 避免「手指张开」；用遮挡/自然姿态；必要时用智能画布局部重绘 |
| 穿模/肢体扭曲 | 复杂动作、人物互动、大动态场景 | 降低动作幅度；使用首尾帧控制；Seedance 2.0 Fast 选择「中等」运镜幅度 |
| 风格漂移 | 提示词中风格词不统一、未固定光线/场景 | 同场景使用固定模板词；批量生成时统一风格前缀；使用参考图锁定风格 |
| 角色「换脸」 | 未正确上传参考图或提示词未重申一致性 | 每镜上传参考图；提示词首句写「参考上传角色形象，保持长相、发型、服装完全一致」 |
| 动作僵硬 | 动作描述抽象 | 用具体动词+幅度，如「眼睛微微睁大、眉毛轻微上扬、身体微微前倾」 |

> 来源 [今日头条](https://www.toutiao.com/article/7533988614884147751/) 提示：目前即梦 AI 在人物一致性方面仍有不足，但会越来越强，建议通过参考图而非仅靠提示词实现一致性。  
> 来源 [CSDN 对比](https://blog.csdn.net/u014177256/article/details/157644721) 指出：即梦提示词跟随得分仅 3.56，存在「镜头不动、动作消失」等问题，需人工筛选与重抽。

## 最佳实践

- **先定角色 ID 再出分镜**：先生成最满意的角色正面图，再作为参考图批量生成分镜。  
- **模型选择**：静态分镜优先使用 Seedream 4.0；动态视频优先使用 Seedance 2.0 Fast。  
- **比例选择**：短视频/短剧分镜常用 9:16 竖屏；角色定妆可用 1:1。  
- **负面提示词**：在图生视频中加入「画面抖动、人物变形、脸部扭曲、动作僵硬、画面闪烁、背景扭曲」。  
- **候选图策略**：每个镜头生成 2–4 张候选图，人工挑选最优，避免「抽一用一」。  

## 避坑

- **参考强度拉满导致姿势呆板**：参考强度高于 80% 容易让角色动作僵硬。  
- **4.0 无限出图但忽视细节**：即使 4.0 一致性强，复杂多人场景仍需分别上传参考图。  
- **直接即梦文生视频跳过静态分镜**：文生视频可控性低于图生视频，短剧建议先出分镜图再转视频。  
- **忽视版权**：避免使用知名 IP、名人肖像，剪映音效/字体选择免费可商用版本。

# 方向3：可灵（Kling）在生图/图生视频中的位置

## 核心定位

可灵（Kling AI）是快手旗下的 AI 视频生成平台，在短剧/漫剧工作流中**既可以直接生图，也可以做图生视频**，但更常被用作**「高质量图生视频 + 长镜头叙事」**环节：

- **可灵图片 3.0**：支持文生图/图生图，最高 4K 分辨率，新增系列组图功能，可批量生成风格一致的画面。  
- **可灵视频 3.0 / 3.0 Omni**：以图生视频为主，支持最长 15 秒连续视频，支持智能分镜、原生音画同步、主体参考。  
- **与即梦的衔接**：通常先用即梦/豆包生成静态分镜图，再导入可灵做图生视频，以获得更稳定的物理运动和电影级质感。

> 来源 [知乎](https://zhuanlan.zhihu.com/p/2002832521004590013) 说明：可灵 3.0 系列包括可灵视频 3.0、可灵视频 3.0 Omni、可灵图片 3.0 及可灵图片 3.0 Omni 四款产品，覆盖静态图像生成、动态视频生成到后期编辑的全链路。  
> 来源 [Kling AI 官网](https://kling-ai.com/app/image/new) 说明：平台支持 video generation、image creation 和 advanced editing capabilities。

## 可灵是否直接生图？还是只图生视频？

| 能力 | 是否支持 | 说明 |
|------|----------|------|
| 文生图 | ✅ 支持（可灵图片 3.0） | 最高 4K 分辨率，强调纹理细节与光影质感。 |
| 系列组图 | ✅ 支持 | 批量生成逻辑连贯、风格统一的画面，适配影视预演与漫画创作。 |
| 图生视频 | ✅ 核心能力 | 可灵 3.0 主打图生视频 + 主体参考。 |
| 文生视频 | ✅ 支持 | 但短剧/漫剧创作者更常用「先图后视频」链路。 |
| 视频编辑/续写 | ✅ 支持 | 支持多轮扩展、首尾帧、智能分镜。 |

> 来源 [知乎](https://zhuanlan.zhihu.com/p/2002832521004590013)：可灵图片 3.0 新增「系列组图」功能，使静态图片具备连环画式叙事能力。

| 灵动画布（Canvas） | 提供多模态编辑与可视化创作空间 | [Kling 官网](https://klingai.com/canvas) |
| 4K 图像生成 | 可灵图片 3.0 最高输出 4K | [知乎](https://zhuanlan.zhihu.com/p/2002832521004590013) |

## 可灵与即梦的衔接：先图后视频

典型组合链路：

```
豆包/DeepSeek 剧本  →  豆包/即梦 生成分镜图  →  可灵 图生视频（高质感）  →  剪映 合成
```

或更细分的组合：

```
即梦 快速出分镜图 + 角色一致性  →  可灵 图生视频（物理运动/电影级质感）  →  剪映 配音/字幕/剪辑
```

> 来源 [知乎教程](https://zhuanlan.zhihu.com/p/1890369960041228029) 结尾提到：生成好的图片可以用可灵去生成视频。  
> 来源 [CSDN 对比](https://blog.csdn.net/u014177256/article/details/157644721) 建议专业团队组合工作流：「Vidu 生成分镜 → 可灵生成高质量视频 → 即梦/剪映后期编辑与分发」。

## 最佳实践

- **可灵更适合「精品镜头」而非「批量素材」**：单个镜头可能需 30–40 次抽卡，成本较高，建议只在关键镜头使用。  
- **先图后视频**：即梦/豆包出分镜图，再导入可灵做图生视频，可获得更好的可控性和一致性。  
- **利用主体参考**：上传角色定妆照或 3–8 秒视频建立「数字演员」，跨场景保持同一张脸、同一个声音。  
- **智能分镜控制**：用自然语言描述「全景推近到特写」或「机位从左向右横移」，减少抽卡次数。  
- **长镜头优先**：可灵原生 15 秒连续生成适合情绪长镜头，避免频繁剪辑打断节奏。  

## 避坑

- **成本爆炸**：可灵成本约为即梦的 10 倍，全片使用可灵会导致预算失控。  
- **速度很慢**：高峰期 5 秒视频可能排队 15 分钟，2 分钟视频需 10–15 分钟，不适合高频量产。  
- **学习曲线陡峭**：功能分散在网页端/APP/小程序，新手上手门槛高于即梦。  
- **不要对中文氛围抱有过高期待**：可灵对中文文化细节的理解不如即梦接地气。

# 方向4：剪映在图片/素材环节的作用

## 核心定位

剪映在短剧/漫剧工作流中**不直接承担「生图」主力**，而是作为**素材整合、后期包装、生态分发**的枢纽：

1. **素材库直接调用即梦资产**：剪映「素材库」已嵌入即梦 AI 生成的图片、视频、音乐资产，无需下载再上传。  
2. **AI 生图/生视频**：剪映本身也提供 AI 图片生成、AI 视频生成、AI 贴纸生成、AI 音乐、AI 数字人等功能。  
3. **封面/片头片尾/字幕/包装**：对分镜图片和视频进行剪辑、调色、配音、加字幕、做封面、做片头片尾。  
4. **与抖音/红果短剧生态打通**：即梦/剪映/抖音/红果短剧形成「创作-剪辑-分发-变现」闭环。

> 来源 [搜狐/三郎](https://www.sohu.com/a/902537987_120142897) 说明：剪映「素材库」已嵌入「即梦 AI」资产，与官方素材使用方法一致，但需同一账号登录。  
> 来源 [搜狐剪映教程](https://www.sohu.com/a/834878441_121724265) 说明：剪映 7.2.0 已集成 AI 文生图、AI 文生视频、AI 音乐、AI 贴纸、AI 数字人、AI 智能包装、AI 花式文字模版。

| 功能 | 操作路径 | 消耗 | 说明 |
|------|----------|------|------|

| AI 视频生成 | 素材 → AI 生成 → 视频生成 | 单次 80 积分 | 支持文本/图片生视频，可设运动速度、运镜、时长。 |

## 剪映做封面、做片头片尾的图片处理

| 环节 | 剪映功能 | 关键参数/技巧 |
|------|----------|---------------|
| 封面 | AI 生图/即梦素材 + 文字模版 + 智能包装 | 突出冲突/钩子，大字标题，人物特写 |
| 片头片尾 | 视频生成 + 文字模版 + 转场 + 音效 | 3 秒内出现片名，片尾留悬念/引导关注 |
| 调色统一 | 调节 → 应用到全部 | 亮度+5、对比度+3、锐化+10、饱和度+2 |
| 字幕 | 智能字幕/识别字幕 | 修正识别错误，选择现代/职场/极简字体 |
| 配音 | 文本朗读 / AI 配音 | 按角色性格选音色，旁白用解说男声/叙事女声 |
| 音效/音乐 | 音频 → AI 音乐 / 音效库 | 情绪关键词如「紧张 职场 轻快」「反转 戏剧性 高潮」 |
| 导出 | 导出 → 1080p / 30fps / MP4 | 勾选高清导出，码率选中/高 |

> 来源 [知乎全流程指南](https://zhuanlan.zhihu.com/p/2008560818791407761) 说明：剪映导出设置建议分辨率 1080P、帧率 30fps、比特率中等、格式 MP4。

## 最佳实践

- **利用「即梦资产库」减少下载上传步骤**：同一账号登录后，直接在剪映素材库中调用即梦图片/视频。  
- **封面做「3 秒钩子」**：人物特写 + 大字号标题 + 高对比配色。  
- **调色统一**：先调好第一个镜头，再点击「应用到全部」，避免各镜头色调不统一。  
- **字幕与口型允许 0.2–0.5 秒误差**：观众注意力在对白内容上，无需逐帧精确对口型。  
- **音效宁少勿多**：3 分钟短剧控制在 8–12 个音效，过多显得杂乱。  

## 避坑

- **账号不一致导致无法调用即梦资产**：确保剪映与即梦登录同一账号。  
- **剪映 AI 生图质量不如即梦 4.0**：关键分镜图建议在即梦中完成，剪映 AI 生图更适合素材/贴纸/封面辅助。  
- **剪映 AI 视频 80 积分/条成本不低**：大量动态镜头仍建议走即梦/可灵生成后再导入剪映。  
- **忽略版权**：剪映部分字体/音效需会员或商用授权，商业发布前需确认。

## 每个工具的强项与弱项（更详细）

### 豆包
- **强项**：短剧剧本生成、分镜脚本拆解、角色设定文本输出、与即梦/剪映同属字节生态。  
- **弱项**：最终图像质量与可控性不如即梦；不能直接做视频。  

### 即梦 AI
- **强项**：
  - 生成速度最快，成本最低，适合高频量产；
  - 中文语义理解精准，操作门槛低；
  - 角色一致性工具（角色特征、参考图、角色 ID、4.0 局部锁定）成熟；
  - 与剪映、抖音、红果短剧生态深度整合，形成「创作-剪辑-分发-变现」闭环；
  - 4.0 支持一次出 14 张系列图、多视角、一句话 P 图。  
- **弱项**：
  - 物理模拟和复杂动作自然度弱于可灵，易穿模；
  - 单次视频时长有限（常规 5–12 秒，标准模式 36 秒）；
  - 提示词跟随得分一般，存在「镜头不动、动作消失」现象；
  - 电影级质感逊于可灵。  

### 可灵 AI
- **强项**：
  - 技术综合实力最强，支持最长 3 分钟连续视频；
  - 物理模拟领先（水流、布料、头发、重力）；
  - 电影级画质，人物微表情与情绪控制精准；
  - 智能分镜、主体锚定、数字演员、原生音画同步；
  - 支持粤语/四川话等方言口型。  
- **弱项**：
  - 生成速度最慢，高峰期排队可达数小时；
  - 成本最高，不适合全片使用；
  - 操作分散在网页/APP/小程序，学习曲线陡峭；
  - 对中文文化氛围理解不如即梦。  

### Midjourney
- **强项**：
  - 艺术创作领域标杆，风格多样、光影细节细腻；
  - 复杂场景全局控制能力强，适合电影分镜/概念设计；
  - 社区资源丰富。  
- **弱项**：
  - 国内用户需解决网络访问；
  - 中文提示词支持弱，需掌握结构化英文 Prompt；
  - 不能直接生成视频，需配合其他工具；
  - 订阅成本高（30 美元/月起），且不支持中文小字体/电商海报等实用场景。

## 推荐组合工作流

### 平民量产流（成本最低）
```
DeepSeek/豆包 写剧本 + 豆包 拆脚本 + 即梦 4.0 生分镜图 + 即梦 Seedance 图生视频 + 剪映 合成
```

### 精品短剧流（质量优先）
```
DeepSeek 写剧本 + 豆包 定角色 + 即梦 4.0 生分镜图（保角色一致） + 可灵 3.0 图生视频（保物理/质感） + 剪映 配音/调色/合成
```

### 漫剧专属流
```
豆包 写古风/女频剧本 + 豆包 拆 5–10 秒镜头 + 即梦/豆包 生角色三视图 + 即梦 4.0 批量生分镜 + Seedance 图生视频 + 剪映 古风 BGM/配音/特效
```

> 来源 [CSDN 对比](https://blog.csdn.net/u014177256/article/details/157644721) 建议专业团队组合：可灵出精品镜头 + 即梦出量产素材 + Vidu 出风格化片段；或 Vidu 生成分镜 → 可灵生成高质量视频 → 即梦/剪映后期编辑与分发。

## 最佳实践

- **工具组合优于单工具**：用豆包做文本、即梦做批量分镜、可灵做精品镜头、剪映做后期，是成本与质量的最优平衡。  
- **优先用「先图后视频」**：无论即梦还是可灵，图生视频的可控性都远高于文生视频。  
- **角色一致性从剧本阶段开始**：角色设定越具体，后续参考图锁定越稳。  
- **根据平台生态选择**：抖音/红果短剧创作者优先即梦+剪映；快手生态创作者可考虑可灵+快影。  
- **成本控制**：可灵用于 10% 的关键镜头即可，90% 的常规镜头用即梦。  

## 避坑

- **全片使用可灵**：成本会飙升到不可接受，且速度极慢。  
- **只看技术指标忽视生态**：短剧最终要分发，剪映/抖音/红果的闭环比单点画质更重要。  
- **Midjourney 做国内短剧分镜**：虽然画质好，但中文支持弱、无视频能力、访问门槛高，不适合国内短剧量产。  
- **忽略「抽卡」成本**：可灵和即梦都需要多轮生成，预算中需预留 20–30% 的重抽成本。  
- **角色设定不够具体**：这是后续「脸崩」的根源，务必在豆包阶段把发型、服饰、道具、特征写细。
